## 二、预测方法论（三层架构）

### 2.1 T1层：逐个SKU时间序列建模（TOP 27个）

**适用条件**：月均销量≥62件，有≥12个月连续销售记录

**方法**：**季节性指数加权法**

```
forecast(SKU, m) = base_avg × seasonality_2024_2025 × trend_factor × weight_recent
```

| 因子 | 计算方式 | 用途 |
|------|---------|------|
| **base_avg** | 过去6个月月均销量 | 捕捉当前水平 |
| **seasonality** | 同月份历史占比指数（24-25年同月/全年均值） | 节日/季节波动 |
| **trend_factor** | 近3月 vs 前3月的增长率 | 上升/下降趋势 |
| **weight_recent** | 最近3个月权重=0.6，之前3个月=0.4 | 近期数据重要性更高 |

**示例** — PISTACHIO 2 STRIPED 26寸粉色（月均193件）：
```
base_avg = (1,018 + 964 + 833 + 884 + 750 + 538) / 6 = 831
seasonality_7月 = 2024年7月 ÷ 2024年月均 = 1.12（7月是旺季）
trend_factor = 近3月均(725) ÷ 前3月均(938) = 0.77（下降趋势）
weight_recent = 0.6

7月预测 = 831 × 1.12 × 0.77 × 0.6 + 831 × 1.12 × 0.4 = 798件
```

### 2.2 T2层：按系列分组+比例分配（39个）

**适用条件**：月均销量21-61件，有销售记录

**方法**：先预测系列总量，再按历史比例分配到SKU

```
系列总量 = 系列内T2+T3 SKU汇总 → 用T1方法预测
SKU分配 = 系列总量 × (该SKU历史占比)
```

**分组逻辑**（按货品编号前缀）：

| 系列前缀 | 代表产品 | 包含SKU数 | 占T2比例 |
|---------|---------|:--------:|:-------:|
| CP2ST_ | PISTACHIO 2 STRIPED | ~15个 | 主力 |
| CBB2_ | PISTACHIO STRIPED | ~8个 | 经典款 |
| DTFB_ | TRUFFLE BACKPACK | ~6个 | 包袋 |
| DMCB_ | MYCENA | ~5个 | 包袋 |
| CPSSD_ | PISTACHIO STRIPED TRUNK | ~5个 | 行李箱 |

### 2.3 T3层：聚合预测+等比例分配（97个）

**适用条件**：月均销量<21件，多为长尾/低频/赠品行

**方法**：全部T3汇总为一个整体 → 用简单移动平均 → 按SKU历史占比分配

```
T3总量 = Σ(T3_SKU_月均) × 季节性系数
SKU_i = T3总量 × (SKU_i_月均 / T3_total_月均)
```

> ⚠️ **T3层尤其需要注意**：这97个SKU中包括了大量赠品（行李牌、贴纸等）和配件，它们的销量与主商品购买量挂钩，实际上有内在关联。但作为第一阶段简化，先按比例分配。

## 三、系统架构设计

### 3.1 模块结构

```
FCST渠道预测看板系统/
├── forecast_system/                  # 预测系统根目录
│   ├── config.py                     # 配置文件（SKU分层、季节系数、参数）
│   ├── data_loader.py                # 数据加载器（读源xlsx，聚合月级数据）
│   ├── data_matcher.py               # SKU匹配器（关联预测SKU与源数据）
│   ├── forecast_engine.py            # 预测引擎（三层方法论实现）
│   ├── output_writer.py              # 输出写入器（写回预测模板）
│   ├── run_forecast.py               # 🔧 主入口（一键运行）
│   └── calibration/
│       ├── accuracy_tracker.py       # 准确性跟踪（与实际对比，自动调参）
│       └── adjustment_log.csv        # 参数调整历史
│
├── 天猫预测-2026年销量预测.xlsx       # 预测模板（输入+输出）
└── 各渠道销售数据源/                 # 源数据仓库
```

### 3.2 数据流

```
[源数据] → data_loader.py → [月度聚合表] → data_matcher.py
                                                      ↓
                                              [SKU×月份矩阵]
                                                      ↓
                                              forecast_engine.py
                                               ├─ T1: 逐个时间序列
                                               ├─ T2: 系列分组分配
                                               └─ T3: 聚合+比例分配
                                                      ↓
                                              output_writer.py
                                                      ↓
                                              [预测模板已更新]
```

### 3.3 高精度策略核心保障

**策略1：数据刷新自动关联**
- 每次运行自动扫描 `各渠道销售数据源/26-*.xlsx`
- 提取直销_伊稻_电商_天猫ITO旗舰店的数据
- 自动匹配预测表B列的163个SKU
- 填充1-6月实际值（作为预测的基线）

**策略2：季节性因子自动计算**
- 用2024、2025两年完整数据，逐月计算季节性指数
- 公式：`Seasonality[month] = 历史该月总销量 / 历史月均总销量`
- 示例：若2024年7月=12,000、2025年7月=13,500，历史月均=10,000
- 则 `Seasonality[7月] = (12000+13500) / (10000*2) = 1.275`（7月是旺季，比平均高27.5%）

**策略3：趋势衰减加权**
- 给近期数据更高权重，远期更低
- 权重方案：`6月权重=0.30, 5月=0.25, 4月=0.20, 3月=0.15, 2月=0.10, 1月=0.05`
- 总和=1.0，越近权重越大

**策略4：新品特殊处理**
- 27个零销售SKU中，18个是2026年度新品（含CPSSD2604、CPSPS2605等）
- 新品预测方案：
  * 有同系列老品的 → 按老品同规格×新品溢价系数（建议1.0~1.2）
  * 全新品类的 → 按首月=0，次月=类似规格老品×0.3，逐步爬坡
